# 小熊记账

一个基于 Web 的个人记账应用，采用前端静态页面 + IndexedDB 本地存储的方式实现，兼容桌面端和移动端，支持离线使用，并可作为渐进式 Web App（PWA）安装到手机主屏幕。

## 1. 项目简介

小熊记账是一个简洁、轻量、可离线使用的个人财务记录工具，适合日常收支管理、家庭记账、个人消费统计等场景。项目目标是在不依赖复杂后端服务的前提下，让用户快速体验到â记录账单、查看统计、管理数据â的完整流程。

当前版本已经具备以下能力：
- 记录收入和支出
- 查看当月账单和历史记录
- 查看收入/支出汇总与统计分析
- 支持本地数据导出与清空
- 支持离线缓存与安装到主屏幕

## 2. 技术架构

### 2.1 前端技术
- HTML：用于搭建应用主体结构
- CSS：用于实现移动端适配、卡片式布局、底部导航、模态弹窗等界面效果
- JavaScript：用于实现页面渲染、表单交互、数据逻辑、统计分析与状态管理

### 2.2 数据存储
- 使用浏览器本地数据库 IndexedDB 存储数据
- 数据包括：
  - 交易记录
  - 分类数据
  - 预算数据
  - 设置项
- 优点：
  - 无需后端部署
  - 支持离线使用
  - 数据保存在本地浏览器中，访问速度快

### 2.3 PWA 能力
- 通过 manifest.json 提供应用安装能力
- 通过 service worker 实现静态资源缓存与离线访问
- 支持添加到手机主屏幕，提升类似原生应用体验

### 2.4 设计特点
- 移动端优先（Mobile First）
- 采用底部标签栏 + 浮动添加按钮的操作方式
- 适配常见手机屏幕尺寸，支持安全区域样式
- 交互简洁，适合日常快速记账

## 4. 项目文件说明

- index.html：应用主页面结构，包含首页、账单、统计、设置四个主要视图
- css/app.css：整体样式文件，负责移动端布局、卡片、按钮、模态框、底部导航等样式
- js/app.js：前端业务逻辑，处理界面渲染、表单提交、导航切换、统计展示与设置操作
- js/db.js：IndexedDB 数据层，封装数据库连接、初始化、增删改查、统计查询等能力
- manifest.json：PWA 配置文件，用于控制应用名称、主题色、图标和安装行为
- sw.js：Service Worker 文件，用于缓存静态资源并支持离线使用
- icons/：应用图标资源

## 5. 数据流说明

应用的数据流大致如下：
1. 用户在界面中填写一条记录
2. 前端调用 JavaScript 逻辑处理数据
3. 数据通过 IndexedDB 写入本地数据库
4. 页面重新渲染，展示最新结果
5. 统计页面从数据库中读取汇总信息并生成图表/排行

这种设计适用于“轻量、私密、无需后端”的场景。

## 6. 本地运行方式

由于这是一个静态前端项目，通常有两种方式运行：

### 方式一：直接打开
- 直接双击打开 index.html
- 适合快速预览，但某些浏览器对本地文件访问的行为可能略有差异

### 方式二：使用本地服务器
推荐使用任意静态文件服务器运行，例如：
- Python 简易服务器：
  - python -m http.server 8000
- 然后在浏览器访问：
  - http://localhost:8000

## 7. 手机端访问说明

如果希望在手机上访问本项目，可以采用以下方式：
- 电脑和手机连接到同一个 Wi-Fi 网络
- 在电脑上启动本地服务器
- 手机浏览器访问电脑的局域网 IP，例如：
  - http://192.168.x.x:8000

如果你希望让手机端也能长期访问，下一步通常需要部署到公网服务器或 GitHub Pages 等静态托管平台。

## 8. 数据存储特点

当前版本的数据是“本地存储”而不是“云端同步”：
- 数据保存在当前设备浏览器的 IndexedDB 中
- 不同设备之间不会自动同步
- 同一浏览器不同场景下也会有各自独立的数据副本

这意味着：
- 适合个人本地使用
- 不适合多设备实时共享
- 如果需要云同步，后续需要接入后端数据库或云服务

## 12. GitHub Pages 更新流程

如果项目已经部署到 GitHub Pages，后续更新代码时可以按照下面流程操作：

### 11.1 进入项目目录
```bash
cd d:\desk\BILLAPP
```

### 11.2 查看当前修改
```bash
git status
```

### 11.3 添加并提交修改
```bash
git add .
git commit -m “更新说明或功能”
```

### 11.4 推送到 GitHub
```bash
git push origin main
```

### 11.5 等待 Pages 自动部署
- 推送成功后，GitHub 会自动开始部署
- 通常几分钟内即可更新到 Pages 网站
- 如果页面没有立即生效，可以刷新浏览器，必要时按 Ctrl + F5 强制刷新

### 11.6 如果页面没有更新
可以检查以下几点：
- 确认 GitHub 仓库里已经看到最新提交
- 确认 GitHub Pages 的部署状态是否成功
- 确认当前使用的分支是否是 main（或 gh-pages）
- 若使用的是 gh-pages 分支，则需要执行：
```bash
git push origin gh-pages
```
一条最简单的命令顺序
git add .
git commit -m “update”
git push origin main
